# Contributing to freee-mcp

freee-mcp への関心とフィードバックをお寄せいただきありがとうございます！
Thank you for your interest in freee-mcp!

## コントリビューションの方針 / Contribution Policy

freee-mcp では Issue ベースのコントリビューションを採用しています。
freee-mcp adopts an issue-based contribution model.

### 歓迎していること / What We Welcome

- バグ報告 / Bug Reports: 再現手順を含めた Issue の作成 / File an issue with reproduction steps
- 機能要望 / Feature Requests: ユースケースを含めた提案 Issue の作成 / File a proposal issue with use cases
- フィードバック / Feedback: 使い勝手や改善点についてのコメント / Comments on usability and improvements
- 質問・議論 / Questions: Issue での質問 / Ask via Issues

### Pull Request について / About Pull Requests

外部からの Pull Request は受け付けておりません。
Pull requests from external contributors are not accepted.

コードの変更を提案する場合は、まず Issue を作成してください。
If you would like to propose a code change, please create an issue first.

これは、プロジェクトの品質と一貫性を維持するための方針です。
This policy is in place to maintain project quality and consistency.

## AI を活用した開発フロー / AI-Powered Development Flow

freee-mcp では AI を活用した開発フローを採用しています。
freee-mcp uses an AI-powered development workflow.

1. Issue として提案・報告を受け取る / Receive proposals and reports as issues
2. メンテナーが Issue をもとに実装方針を検討する / Maintainers review and plan implementation based on issues
3. AI を活用して Issue ベースで実装する / Implement based on issues using AI

この仕組みにより、コードの一貫性と品質を保ちながら迅速に開発を進めています。
This approach allows us to develop rapidly while maintaining code consistency and quality.

## Issue の書き方 / How to Write Issues

Issue を作成する際は、テンプレートに沿って記入してください。
When creating an issue, please follow the provided templates.

- [Bug Report / バグ報告](https://github.com/freee/freee-mcp/issues/new?template=bug_report.md)
- [Feature Request / 機能要望](https://github.com/freee/freee-mcp/issues/new?template=feature_request.md)
